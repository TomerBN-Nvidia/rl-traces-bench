## Goal

Build a reproducer to *measure ourselves* on the RL rollout long-tail problem:
take (or synthesize) a trace of per-turn ISL/OSL + prefix caching and replay it as
a multi-turn agentic benchmark, so we can quantify the tail and A/B approaches
(MTP, chunked prefill, routing, …). Check first whether `aiperf` can replay such a
trace; if yes, create the traces. Deliverable: a clean PR in the cheat-sheet.

## Logic (how the non-obvious pieces work)

- **OSL quantile sampler (`distributions.py`).** The real OSL distribution is
  bimodal (80% short + a heavy tail), so we do **not** fit a parametric family.
  Instead we build an inverse-CDF from the published (quantile, tokens) anchor
  points and interpolate between them in **log-token space**; sampling `u∈[0,1]`
  through it reproduces the anchors by construction. This is why the token-domain
  validation passes trivially at large N.

- **ISL growth + cumulative prefix `hash_ids` (`prompt_model.py`).** Turn *t*'s
  prompt = system + Σ(prior user msgs + prior assistant outputs) + this turn's
  user msg → ISL is monotonically non-decreasing. `hash_ids` block-align the ISL
  (`ceil(ISL/block_size)`); the first `shared_blocks` are global (system prompt,
  cached across the batch), the rest are per-rollout (`rollout_base+idx`,
  `rollout_base=(sid+1)*1_000_000`). Each turn's block set is a **superset** of
  the previous → real within-session prefix-cache hits; disjoint per-rollout
  ranges → no false cross-rollout sharing.

- **Static-batch metrics (`metrics.py`).** A session (rollout) completes at its
  last turn's end; makespan = max over sessions; **tail bubble = makespan − p90**
  (the wait-on-tail idle time); **goodput proxy = mean/makespan**.

- **Dual validation gate (`analyze.py`).** Token domain: realized OSL percentiles
  vs published anchors. Time domain: the *served* completion distribution's SHAPE
  ratios (p99/p50, max/p50) vs the real `nemorl_trace` shape — ratios, not
  absolute times (those are model/config-dependent).

- **Tau2 token↔time estimator + the Gauss solver (`tau2_estimator.py`).** Fits
  `duration ≈ a + b·ISL + c·OSL` by ordinary least squares: build the normal
  equations `AᵀA` (3×3) and `Aᵀb` (3-vector) from feature rows `x=[1, ISL, OSL]`,
  then solve the 3×3 system with **Gauss-Jordan elimination + partial pivoting**
  — for each pivot column swap in the row with the largest |pivot| (numerical
  stability), normalize the pivot row, eliminate that column from all other rows;
  after 3 passes the augmented matrix is the identity and its last column is the
  solution `[a, b, c]`. `invert_duration` then solves `OSL = (dur − a − b·ISL)/c`.
  All stdlib (no numpy) so it runs in the laptop/login env. Real-data fit is
  deferred to HSG (that's the Tau2 data the user asked to defer).

## Bugs hit and how they were fixed

The per-task + final reviews caught **6 real bugs — 5 of them in the plan's own
example code** (a good argument for reviewing plan-generated code, not trusting
it):

1. **`split_osl` infinite loop when `total < turns`** (Task 2, fix `1377937`).
   Every part floored to 0 then pinned to 1 via `max(1,0)`, so `sum > total`,
   `diff` went negative, but the decrement branch (`parts[j] > 1`) could never
   fire → non-terminating `while`. With turn counts 15–30 and small OSL totals
   this was reachable and would hang the reproducer. Fix: feasibility clamp
   `total = max(total, turns)` (a rollout with T turns can't emit <T tokens) +
   edge tests (total<turns, total==turns, turns==1/total==0).

2. **`hash_ids_for` silently violated its superset guarantee** on non-monotonic
   `isls` (Task 3, fix `c82e0a1`). The whole point of the function is a cumulative
   prefix, but it accepted any list. Fix: guard raises `ValueError` if `isls`
   isn't non-decreasing + a negative test. (Safe on today's only caller —
   `per_turn_isl` is monotonic by construction.)

3. **`per_rollout` stats duplicated per turn** (Task 4, fix `106b8d5`). The stats
   append `all_osl.append(sum(osls))` sat inside the per-turn loop, so the rollout
   total was added ~15–30× → percentiles biased toward high-turn rollouts. Fix:
   append the rollout total once per rollout (outside the turn loop); per_turn
   path unchanged; added a test that reconstructs rollout totals from the records.

4. **`validate_time_domain` test was mathematically impossible** (Task 6, fix
   `61ef24a`). The brief asserted `validate_time_domain([84,909,1669])` passes,
   but nearest-rank p50 of a 3-element list is the **middle** element (909), not
   84 — you can't encode p50=84/p99=909/max=1669 in 3 points. The implementer
   correctly refused to fudge it and reported DONE_WITH_CONCERNS. Fix: a valid
   100-point distribution `[84]*51+[909]*48+[1669]` (p50=84, p99=909, max=1669) +
   a flat-distribution negative case. Also `11650b3`: a scoped `.gitignore` negate
   for the test fixture (repo root blanket-ignores `*.jsonl`).

5. **`EXTRA_SERVE_ARGS` unquoted expansion corrupted the MTP config** (Task 7, fix
   `ef3cf4f`). `configs/mtp.env` carried `--speculative-config '{"method":...}'`
   as a string, expanded **unquoted** in the serve script → the literal single
   quotes became part of the argv token → invalid JSON, broken MTP run. Statically
   reproducible (not a cluster unknown). Fix: make `EXTRA_SERVE_ARGS` a **bash
   array** in every config and expand `"${EXTRA_SERVE_ARGS[@]}"` in the serve
   script; verified the JSON survives as one clean token.

6. **A/B compare seam gap: `report.json` was never written** (final whole-branch
   review, fix `163335c`). `analyze.main()` wrote only `report.html` and *printed*
   JSON, but `compare.py` reads `report.json` — so the documented A/B step died
   with `FileNotFoundError`. This is a DoD item. Fix: add `--out-json` to
   `analyze.py`, wire it through `run_batch.sh` + README, + a test that exercises
   the JSON artifact `compare.py` depends on. Also dropped an unused import and
   corrected an over-claim in the README about auto-scraping prefix-cache hits.

Each fix was re-reviewed and synced back into the plan document so the plan can't
reintroduce the bug.

## Lessons / notes

- **The HTMLs had no tokens.** Anyone expecting to "replay the recorded ISL/OSL"
  should know: only the OSL *distribution* (published percentiles) and the turn
  *skeleton* + *durations* are real; per-turn ISL is modeled; real per-turn
  (ISL,OSL) pairs must be captured on HSG (Tau2 export or an instrumented aiperf
  run) — that's what `tau2_estimator.py` is for.
- **Static batch is the right MVP framing**: makespan − p90 *is* the long-tail
  bubble, no RL loop required; it maps directly onto Phase 2's lag-1 fixed batch.
- **Reviewing plan-generated code pays off** — 5 of 6 bugs were in the plan's own
  snippets, invisible to a single happy-path test.
- **Keyword scans over minified HTML lie**: early `grep isl/osl/hash` counts were
  polluted by the Plotly library (`isDark`, `frameHash`); always parse the real
  data blob (see `scripts/probe_traces.py`).
- Execution locality is a hard rule here: nothing aiperf-related runs locally; the
  mock smoke also runs on HSG.
